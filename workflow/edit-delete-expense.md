Sometimes an expense is logged with the wrong amount, wrong category, or wrong date. Users need to be able to fix or remove entries from both the app and directly in the Google Spreadsheet.

## User Stories

- As a user, I want to edit a past expense so I can correct a mistake without deleting and re-entering it
- As a user, I want to delete an expense so I can remove a duplicate or an entry that shouldn't exist
- As a user, I want changes made in the app to be reflected in the Google Spreadsheet immediately
- As a user, I want changes made in the Google Sheet to be reflected in the app immediately

## Success

- Tapping an expense in the today list or History opens an edit screen pre-filled with its values
- All fields are editable: amount, category, date, paid by, notes
- Either user can edit or delete any expense regardless of who created it
- Saving updates the row in the Expenses tab
- Deleting removes the row from the Expenses tab with a confirmation dialog
- Changes made directly in the Google Spreadsheet are reflected in the app on next read
- Editing or deleting an expense triggers a refresh of reports and spending insights (entity 014)
- Auto-generated subscription expenses can be edited or deleted — with a warning reminding the user to update the subscription directly to prevent the same entry from recurring

### Out of Scope

- Bulk edit or delete
- Undo after delete
- Edit history or audit log

## Plan

### Entry Point

From the today list (Home) or History: tap any expense row → opens edit screen.

### Edit Screen

Same layout as the entry form (entity 002), pre-filled with the expense's current values. Either user can open and edit any expense.

"Save" updates the row. "Delete" shows a confirmation dialog.

### Subscription Expense Warning

If the expense was auto-generated by a subscription (`subscription_id` is set), show a banner at the top of the edit screen:

> "This expense was created by a subscription. Editing it here won't affect future entries — update the subscription directly if the amount or details have changed."

The same warning appears in the delete confirmation dialog.

### Confirmation on Delete

Dialog: "Delete this expense?" with Cancel and Delete buttons. No undo.
For subscription expenses, dialog adds: "This was created by a subscription. Future entries won't be affected."

### Writes to Spreadsheet

- Edit: Firebase Function updates the matching row by `id`
- Delete: Firebase Function removes the matching row by `id`

## Decisions

- **Workflow design:** Ideation is a gate — not every observation moves to spec. Captain explicitly said "I would gate at ideation."
- **Tech stack:** Next.js PWA (not native app), Firebase Hosting + Functions + Auth, Google Sheets as single source of truth, DaisyUI fantasy theme.
- **Public repo:** Code is public; credentials stay in `.env` (gitignored); data lives in Google Drive. No sensitive info in the repo.
- **Category longevity:** Categories referenced by `category_id` only. Rename retroactively updates display; archive, never delete. Chosen to handle 20-year horizon.
- **Subscription auto-confirm:** Recurring entries are created as `confirmed` immediately — no pending/review state.
- **Navigation:** Standalone entity (011) defines the shell/frame; each feature entity defines its tab content.
- **History:** Grouped by day with date header and daily total, same visual pattern as today's list on Home.
- **Spec template:** Added User Stories section to feature template per captain's request.
